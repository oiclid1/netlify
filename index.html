Technical Support Activities

  1. Most Favorite Technical Support Activities From List
    • Debug a customer's build using a programming language and framework that I've never seen before
    • Set up my own copy of several static site frameworks for debugging
    • Work with people to figure out if Netlify's service can solve a particular workflow or integration challenge they have
    • Help manage communications during a service outage
    • Analyze thousands of support tickets to spot trends to improve our product

  2. Least Favorite Technical Support Activities From List
    • Deliver a talk to many people I don't know at a conference or meetup
    • Create video tutorials to help teach users a specific feature or use case
    • Respond to Netlify fans on Twitter
    • Engage multiple users at once via chat to answer their questions and troubleshoot problems
    • Receive occasional phone calls requesting support from our highest-value customers

  3. The greatest thing about technical support is that one gets the opportunity to make the product as frictionless as possible. This could be through showing users how to use the product, show them a better way to use the product, and help discover and fix any issues that make using the product hard.

Netlify/Staticgen

I was really impressed with how easy it was to host on Netlify. I have used a lot of hosting services before, but the only one that compare in terms of how easy it is to deploy a site would be GitHUb Pages. The only problem I had was with a couple of the Static Site Generators. The first two ones I chose gave me a little trouble, but once I was able to find one that did not require too much effort to setup, the whole thing was a breeze!

I did not want to spend a lot of time installing modules and dependencies, so I decided to use work with a site generator based on JavaScript. I eventually had to try three different site generators before I found one that worked easily for me. I first tried Gatsby, but I kept getting all sort of error messages, most of which seemed related to out-of-date modules. After I got tired of updating Node, NPM, and eventually individual modules, I just threw in the towel. I decided to try Enduro. I don’t know if it was because my internet speed was slow, or what, but it took way too long to setup. At the end all I got a couple was an Unhandled Promise Rejection message. I cancelled it after waiting for it to end after 30 minutes. I decided to find the simplest JavaScript site generator on staticgen. I needed no bells and whistles, just straight up install and run. And so, I chose Silex. It was so easy to run (I cloned a repo and opened the index.html file) that I felt stupid spending all that time trying to debug other site generators. For simple projects henceforth, I would use Silex.

I currently cannot think of any suggestion on how to improve this test and/or the job posting.

Documentation

My favorite technical product documentation is the Amazon Web Services (AWS) documentation. I love it because, unlike a lot of other product documentations, following the instructions ALWAYS work. At least I cannot remember an instance in which they did not A lot of the time, documentation seems to be written for an OS, product version, or even year other than the one you are using. Or live in. It can be frustrating that one had to spend a lot of time in forums looking for solutions after spending a lot of time going through the documentation. I know there are a lot of products I have not used in AWS, but for all of those I have, AWS’ docs has always had my back in ways that Google Cloud, Heroku, GitHub Pages, GoDaddy, MongoDB, Node, Django, MySQL, etc never have. I think it is the one and only service or software that I have learned how to use by reading only the documentation.

SSL/HTTPS

SSL/HTTPS provides a secure connection between the web host and the client. This is important because it:

  1. Encrypts all information being transferred across the connection. This makes it hard for anybody to easily eavesdrop on the connection. The user is assured that his/her sensitive information such as passwords, card numbers, etc, are secure and not easily stolen.
  2. It provides authentication, assuring the user that information is being sent to (and received from) the right servers, and not to imposters.
  3. It enable users to have greater trust that the websites they are transacting with is not an imposter, and that they are secure.
